# 多Agent钉钉集成架构设计

## 系统概述

基于散户超短线（1-3天）操作需求，设计的多Agent钉钉协同炒股系统。

## 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                   钉钉控制中心                           │
│  • 指令接收与分发                                      │
│  • 消息格式化与推送                                    │
│  • Agent状态监控                                       │
│  • 风险控制与熔断                                      │
└───────────────┬─────────────────────────────────────────┘
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
┌─────────┐ ┌─────────┐ ┌─────────┐
│数据Agent│ │分析Agent│ │交易Agent│
│(采集)   │ │(决策)   │ │(执行)   │
└─────────┘ └─────────┘ └─────────┘
    │           │           │
    └───────────┼───────────┘
                ▼
        ┌───────────────┐
        │  风控Agent    │
        │(监控与预警)   │
        └───────────────┘
```

## Agent详细设计

### 1. 钉钉控制中心 (DingTalk Controller)
**职责**：
- 接收钉钉用户指令
- 协调各Agent工作流
- 格式化并推送消息
- 监控Agent健康状态

**钉钉指令集**：
```
# 查询指令
/热点       # 查看今日热点板块
/分析 002594 # 分析指定股票
/持仓       # 查看当前持仓
/盈亏       # 查看当日盈亏
/状态       # 查看系统状态

# 操作指令
/买入 002594 10% # 买入股票
/卖出 002594 全部 # 卖出股票
/止损 002594 200 # 设置止损价
/目标 002594 240 # 设置目标价

# 配置指令
/模式 激进     # 设置风险模式
/关注 AI 新能源 # 设置关注板块
/限额 5%      # 设置单日亏损限额
```

### 2. 数据采集Agent (Data Collector Agent)
**运行模式**：后台持续运行
**数据源**：
- akshare-finance（实时行情）
- web-search（新闻舆情）
- 自定义数据源

**钉钉推送**：
- 09:15：开盘热点
- 10:00/14:00：资金流向
- 实时：板块异动（>3%）
- 实时：个股涨停/跌停
- 收盘后：龙虎榜数据

### 3. 技术分析Agent (Technical Analysis Agent)
**运行模式**：定时触发（每15分钟）
**分析框架**：
- 多周期：60分钟/日线/周线
- 多指标：MACD/KDJ/RSI/成交量
- 形态识别：突破/反转/持续

**钉钉推送**：
- 买入信号：综合评分≥7.5
- 卖出信号：跌破关键支撑
- 预警信号：指标背离/异常放量

### 4. 信号融合Agent (Signal Fusion Agent)
**运行模式**：事件驱动
**融合规则**：
- 技术面权重：40%
- 资金面权重：30%
- 基本面权重：20%
- 消息面权重：10%

**输出**：
- 五位一体分析表
- 综合评分（1-10）
- 具体操作建议

### 5. 交易执行Agent (Trade Execution Agent)
**运行模式**：指令驱动
**功能**：
- 模拟交易（测试用）
- 实盘监控（如有接口）
- 仓位管理
- 止损止盈执行

**钉钉推送**：
- 交易确认：买入/卖出执行结果
- 持仓更新：实时盈亏
- 风险预警：接近止损/止盈

### 6. 风控监控Agent (Risk Control Agent)
**运行模式**：实时监控
**监控项**：
- 单只股票仓位≤30%
- 单日亏损≤5%
- 连续亏损≤3次
- 交易频率监控

**钉钉推送**：
- 风险预警：超限提醒
- 熔断通知：暂停交易
- 恢复通知：风险解除

## 工作流程

### 日常操作流程
```
08:30-09:00 数据Agent → 准备开盘数据
09:15       钉钉推送 → 开盘热点
09:30-11:30 分析Agent → 实时扫描
            信号Agent → 生成建议
            钉钉推送 → 买卖信号
11:30       钉钉推送 → 午间复盘
13:00-15:00 继续实时监控
15:00       钉钉推送 → 收盘总结
15:00-16:00 风控Agent → 绩效分析
```

### 信号处理流程
```
1. 数据Agent检测到异动
2. 分析Agent进行技术分析
3. 信号Agent综合评分
4. 评分≥7.5 → 生成买入建议
5. 钉钉推送买入信号
6. 用户确认 → 交易Agent执行
7. 风控Agent监控持仓
```

## 风险控制

### 技术风险
1. **数据延迟**：多数据源备份
2. **API限制**：请求频率控制
3. **网络中断**：本地缓存与重试

### 操作风险
1. **过度交易**：频率限制
2. **情绪化决策**：强制冷静期
3. **黑天鹅事件**：熔断机制

### 安全风险
1. **信息泄露**：加密存储
2. **未授权访问**：身份验证
3. **恶意指令**：指令白名单
