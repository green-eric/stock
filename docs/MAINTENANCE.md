# 项目维护指南

## 概述
本文档提供了项目的维护和运维指南，包括日常维护、故障排查、性能优化和系统监控等内容。

## 系统架构

### 技术栈
- **前端**: React 18 + TypeScript + Vite
- **后端**: Node.js + Express + TypeScript
- **数据库**: PostgreSQL 14
- **缓存**: Redis 7
- **消息队列**: RabbitMQ
- **容器化**: Docker + Docker Compose
- **编排**: Kubernetes (生产环境)

### 部署架构
```
负载均衡器 (Nginx)
    ↓
应用服务器集群 (Node.js)
    ↓
数据库集群 (PostgreSQL主从)
    ↓
缓存集群 (Redis哨兵)
```

## 故障排查

### 常见问题及解决方案

#### 1. 应用启动失败
**症状**: 应用无法启动，端口被占用或依赖服务不可用
**排查步骤**:
1. 检查端口占用: `netstat -tulpn | grep :3000`
2. 检查依赖服务: `systemctl status postgresql redis`
3. 查看应用日志: `tail -f /var/log/app/error.log`
4. 检查环境变量: `printenv | grep APP_`

#### 2. 数据库连接问题
**症状**: 数据库连接超时或连接池耗尽
**排查步骤**:
1. 检查数据库状态: `systemctl status postgresql`
2. 检查连接数: `SELECT count(*) FROM pg_stat_activity;`
3. 检查最大连接数: `SHOW max_connections;`
4. 优化连接池配置

#### 3. 性能下降
**症状**: API响应时间变长，CPU/内存使用率升高
**排查步骤**:
1. 监控系统资源: `top`, `htop`, `vmstat`
2. 分析慢查询: `EXPLAIN ANALYZE <query>`
3. 检查缓存命中率
4. 分析应用性能: Node.js profiling

#### 4. 内存泄漏
**症状**: 内存使用持续增长，最终导致OOM
**排查步骤**:
1. 监控内存趋势
2. 生成堆快照: `node --inspect app.js`
3. 分析GC日志
4. 检查循环引用

#### 5. 数据采集失败（多Agent炒股系统）
**症状**: 钉钉收不到市场热点推送，数据文件未更新
**排查步骤**:
1. 检查AKShare API可用性: `python -c "import akshare; print(akshare.__version__)"`
2. 查看数据采集Agent日志: `tail -f logs/data_agent.log`
3. 验证网络连接: `ping api.akshare.com`
4. 检查降级机制是否触发: 查看日志中是否有"使用模拟数据"提示
5. 测试钉钉连通性: `python test_dingtalk_sign_python.py`
**解决方案**:
- 若AKShare不可用，系统会自动降级至模拟数据
- 检查依赖包: `pip install --upgrade akshare pandas`
- 重启数据采集Agent: `./start_agents.sh restart data_agent`

### 紧急恢复流程
1. **识别问题**: 确定故障范围和影响
2. **通知团队**: 立即通知相关技术人员
3. **启动预案**: 执行相应的应急预案
4. **数据恢复**: 如有数据丢失，从备份恢复
5. **根本原因分析**: 故障解决后进行详细分析
6. **预防措施**: 制定防止再次发生的措施

## 性能优化

### 数据库优化
1. **索引优化**
   - 为频繁查询的字段创建索引
   - 定期分析索引使用情况
   - 删除无用索引

2. **查询优化**
   - 避免SELECT *
   - 使用EXPLAIN分析查询计划
   - 优化JOIN操作

3. **连接池优化**
   - 合理设置最大连接数
   - 配置连接超时时间
   - 监控连接池状态

### 应用优化
1. **代码层面**
   - 避免同步阻塞操作
   - 使用异步/await处理I/O
   - 实现请求限流和熔断

2. **缓存策略**
   - 热点数据缓存
   - 缓存失效策略
   - 分布式缓存一致性

3. **静态资源优化**
   - CDN加速
   - 资源压缩
   - 浏览器缓存

### 基础设施优化
1. **负载均衡**
   - 健康检查配置
   - 会话保持策略
   - 流量分发算法

2. **自动扩缩容**
   - 基于CPU/内存的自动扩缩
   - 基于请求量的自动扩缩
   - 预热策略

   - 磁盘使用率 > 80%
   - 缓存命中率 < 80%
