## 快速开始

### 系统要求
- **设备**: 智能手机（安装钉钉）
- **网络**: 稳定的互联网连接
- **权限**: 钉钉群管理员权限
- **时间**: 建议在交易时段使用（09:30-15:00）

### 第一步：钉钉配置
1. **创建钉钉机器人**
   - 打开钉钉群 → 设置 → 智能群助手
   - 添加机器人 → 自定义
   - 设置机器人名称：`多Agent炒股助手`

2. **安全设置**
   - **关键词**: 添加 `买入`、`卖出`、`预警`、`热点`、`测试`
   - **签名**: 启用签名校验，记录Secret
   - **IP白名单**: 可选，增加安全性

3. **获取配置信息**
   - **Webhook地址**: `https://oapi.dingtalk.com/robot/send?access_token=xxx`
   - **签名密钥**: `SECxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

### 第二步：系统启动
```bash
# 进入项目目录
cd .

# 启动系统
./start_agents.sh start

# 查看系统状态
./start_agents.sh status
```

### 第三步：验证功能
1. **检查钉钉消息**
   - 系统启动通知
   - 连通性测试消息

2. **发送测试指令**
   ```
   热点
   状态
   帮助
   ```

3. **确认系统运行**
   - 数据采集Agent: 每5分钟运行
   - 技术分析Agent: 每15分钟运行
   - 系统监控Agent: 每30分钟运行

## 系统功能详解

### 1. 数据采集功能

#### 数据源
- **市场数据**: 热点板块、涨跌幅、成交量（来源：AKShare、新浪财经、东方财富）
- **资金流向**: 北向资金、主力资金、散户资金（来源：AKShare、新浪财经、东方财富，单位自动转换为亿）
- **新闻舆情**: 政策消息、公司公告、行业新闻（规划中）

> **注**: 系统已集成多数据源（AKShare、新浪财经、东方财富），具备智能数据源切换机制。当主数据源不可用时，自动切换至备用数据源，确保系统持续运行。

#### 数据更新频率
- **实时数据**: 每5分钟更新
- **盘后数据**: 收盘后15分钟汇总
- **历史数据**: 每日归档保存

#### 数据查看方式
```bash
# 查看最新市场数据
ls -la ./data/market_*.json

# 查看数据内容
cat ./data/market_20260308_0915.json | head -50
```

### 2. 数据共享功能

#### 功能说明
- **实时数据共享**: 各Agent之间实时共享市场数据、持仓信息和分析结果
- **数据持久化**: 自动保存数据到本地文件，确保系统重启后数据不丢失
- **数据变更通知**: 支持订阅数据变更，实时响应市场变化
- **数据过期检查**: 自动检测数据新鲜度，避免使用过期数据

#### 数据共享内容
- **市场数据**: 热点板块、资金流向等实时市场信息
- **持仓数据**: 当前持仓、交易记录等账户信息
- **分析结果**: 技术分析结果、策略建议等决策信息

#### 数据共享查看
```bash
# 查看共享数据文件
cat ./data/shared_data.json

# 查看数据内容
cat ./data/shared_data.json | head -100
```

### 2. 技术分析功能

#### 分析指标
- **趋势指标**: MA、EMA、MACD
- **动量指标**: RSI、KDJ、CCI
- **波动指标**: BOLL、ATR
- **成交量**: VOL、OBV

#### 分析周期
- **超短线**: 60分钟线、日线
- **短线**: 日线、周线
- **趋势**: 周线、月线

#### 信号生成
```
综合评分算法:
技术面 × 0.4 + 资金面 × 0.3 + 基本面 × 0.2 + 消息面 × 0.1

信号阈值:
买入信号: ≥ 7.5分
卖出信号: ≤ 4.0分
观望信号: 4.0-7.5分
```

#### 分析结果查看
```bash
# 查看最新分析结果
ls -la ./data/analysis_*.json

# 查看具体分析
cat ./data/analysis_20260308_0930.json | head -100
```

### 4. 风险控制功能

#### 仓位管理
- **单只股票**: ≤ 30%仓位
- **总仓位**: ≤ 80%仓位
- **最小仓位**: ≥ 5%仓位

#### 止损规则
- **价格止损**: 下跌8%止损
- **时间止损**: 持有3天未达目标
- **移动止损**: 盈利5%后启动

#### 交易限制
- **每日交易**: ≤ 5次
- **连续止损**: ≤ 3次
- **冷却时间**: 24小时

#### 风险报告
系统会在以下情况发送风险报告：
- 仓位超过限制
- 接近止损位
- 连续亏损
- 市场异常波动

### 6. 错误处理功能

#### 功能说明
- **统一错误处理**: 集中处理系统中的各种错误
- **错误重试机制**: 自动重试失败的操作，提高系统可靠性
- **错误统计分析**: 收集错误信息，便于问题排查
- **错误降级处理**: 当关键功能失败时自动降级到备用方案

#### 错误类型
- **数据采集错误**: 数据源连接失败、数据解析错误
- **分析计算错误**: 技术指标计算失败、分析算法错误
- **网络通信错误**: 钉钉消息发送失败、API调用失败
- **系统资源错误**: 内存不足、磁盘空间不足

#### 错误处理机制
```
1. 捕获错误并记录详细信息
2. 尝试自动重试（最多3次）
3. 如果重试失败，执行降级策略
4. 发送错误通知（可选）
5. 记录错误统计信息
```

## 故障排除

### 常见问题解决

#### 问题1：钉钉收不到消息
**症状**:
- 发送指令无响应
- 系统推送未收到

**排查步骤**:
1. 检查网络连接
2. 检查钉钉机器人状态
3. 验证关键词设置
4. 测试签名配置

**解决方案**:
```bash
# 测试钉钉连通性
python test_dingtalk_sign_python.py

# 检查配置文件
cat ./config/dingtalk.json

# 手动发送测试消息
python send_final_notice.py
```

#### 问题2：Agent停止运行
**症状**:
- 系统状态报告异常
- 数据不更新
- 分析无结果

**排查步骤**:
1. 检查Agent进程
2. 查看日志文件
3. 检查系统资源

**解决方案**:
```bash
# 查看Agent状态
./start_agents.sh status

# 查看日志
tail -f ./logs/*.log

# 重启系统
./start_agents.sh restart
```

#### 问题3：数据分析不准确
**症状**:
- 买卖信号异常
- 评分不合理
- 建议不符合预期

**排查步骤**:
1. 检查数据源
2. 验证分析算法
3. 检查参数配置

**解决方案**:
```bash
# 检查数据文件
ls -la ./data/

# 查看分析结果
cat ./data/analysis_*.json | head -50

# 调整分析参数
vi ./config/dingtalk.json
```

## 常见问题

### Q1: 系统适合什么类型的投资者？
**A**: 系统专为**散户超短线（1-3天）**投资者设计，适合：
- 关注市场热点的投资者
- 喜欢技术分析的投资者
- 需要风险控制的投资者
- 希望自动化辅助的投资者

### Q2: 系统的准确率如何？
**A**: 系统准确率受多种因素影响：
- **数据质量**: 依赖数据源的准确性
- **市场环境**: 不同市场表现不同
- **参数设置**: 需要根据个人风格调整
- **建议**: 先用模拟盘测试，再实盘操作

### Q3: 需要多少资金？
**A**: 系统支持各种资金规模：
- **小资金**: 1-5万元，建议单只股票≤30%
- **中资金**: 5-20万元，建议分散投资
- **大资金**: 20万元以上，建议分批操作

### Q4: 如何提高收益？
**A**: 建议策略：
1. **严格止损**: 亏损8%立即止损
2. **分批建仓**: 不要一次性满仓
3. **关注热点**: 跟随市场主线
4. **控制情绪**: 避免追涨杀跌
5. **持续学习**: 总结经验教训

### Q5: 系统有哪些限制？
**A**: 当前版本限制：
- 数据源依赖外部API
- 分析基于历史数据
- 需要人工最终确认
- 不支持自动交易

### Q6: 如何获得技术支持？
**A**: 支持渠道：
- **钉钉群**: 实时在线支持
- **日志文件**: 自助排查问题
- **文档查询**: 参考使用手册
- **系统报告**: 查看状态信息
