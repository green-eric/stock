## 系统概述

### 系统简介
多Agent炒股系统是一个专为散户超短线（1-3天）操作设计的智能投资辅助系统。系统通过多个协同工作的Agent，结合钉钉实时推送，提供市场热点追踪、技术分析、买卖信号和风险控制等功能。

### 核心功能
- **多Agent协同**: 数据采集、技术分析、风险控制、交易执行等多个Agent协同工作
- **实时数据**: 接入多个数据源，确保数据实时性和准确性
- **智能分析**: 基于机器学习的技术分析和信号生成
- **风险控制**: 完整的仓位管理、止损机制和熔断保护
- **自动化交易**: 支持模拟盘和实盘交易
- **可视化界面**: Web管理控制台，实时查看系统状态和分析结果

### 技术特点
- **微服务架构**: 各组件独立部署和扩展
- **实时性**: 分钟级数据更新和分析
- **安全性**: 多层安全防护机制
- **可扩展性**: 支持水平扩展和功能插件
- **可观测性**: 完善的监控和日志系统

## 系统要求

### 硬件要求
- **CPU**: 至少4核
- **内存**: 至少8GB
- **存储**: 至少50GB
- **网络**: 稳定的互联网连接

### 软件要求
- **操作系统**: Linux (Ubuntu 20.04+) 或 Windows 10+
- **Docker**: 20.04+
- **Docker Compose**: 1.29+
- **Python**: 3.9+ (仅开发环境需要)

### 网络要求
- 能够访问互联网，用于数据采集和钉钉推送
- 能够访问券商交易接口（实盘交易时需要）

## 系统使用

### 1. 钉钉指令

#### 查询指令
- **热点**: 查看市场热点板块
- **分析 [股票代码]**: 分析指定股票
- **持仓**: 查看当前持仓
- **盈亏**: 查看当日盈亏
- **状态**: 查看系统状态
- **帮助**: 查看可用指令

#### 操作指令
- **买入 [股票代码] [仓位]**: 买入指定仓位
- **卖出 [股票代码] [仓位]**: 卖出指定仓位
- **止损 [股票代码] [价格]**: 设置止损价
- **目标 [股票代码] [价格]**: 设置目标价

#### 配置指令
- **模式 [保守/稳健/激进]**: 设置风险模式
- **关注 [板块1] [板块2]**: 设置关注板块
- **限额 [百分比]**: 设置单日最大亏损
- **静默 [开始时间]-[结束时间]**: 设置静默时段

### 2. Web管理界面

#### 访问方式
- **地址**: http://localhost:8000
- **登录**: 使用配置的用户名和密码

#### 主要功能
- **Dashboard**: 系统概览，显示系统状态和关键指标
- **Market Data**: 市场数据，查看热点板块和资金流向
- **Technical Analysis**: 技术分析，查看股票分析结果和买卖信号
- **Trades**: 交易管理，查看交易记录和当前持仓
- **Settings**: 系统设置，配置系统参数和Agent参数
- **Monitoring**: 系统监控，查看系统状态和性能指标

### 3. API接口

#### API文档
- **地址**: http://localhost:8000/docs
- **格式**: OpenAPI 3.0

#### 常用API
- **系统状态**: `GET /api/v1/system/status`

- 🟡 **黄色预警**: 连续止损2次
- 🔵 **蓝色预警**: 仓位超过限制

#### 市场预警
- 📉 **下跌预警**: 板块跌幅>3%
- 💸 **资金预警**: 资金大幅流出

- 🔄 **背离预警**: 指标价格背离

## 故障排除

### 常见问题解决

#### 问题1：钉钉收不到消息
**症状**:
- 发送指令无响应
- 系统推送未收到

**排查步骤**:
1. 检查网络连接
2. 检查钉钉机器人状态
3. 验证关键词设置
4. 测试签名配置

**解决方案**:
```bash
# 测试钉钉连通性
curl -X POST https://oapi.dingtalk.com/robot/send?access_token=xxx \
  -H "Content-Type: application/json" \
  -d '{"msgtype":"text","text":{"content":"测试消息"}}'

# 检查配置文件
cat config/dingtalk.json
```

#### 问题2：Agent停止运行
**症状**:
- 系统状态报告异常
- 数据不更新
- 分析无结果

**排查步骤**:
1. 检查Agent进程
2. 查看日志文件
3. 检查系统资源

**解决方案**:
```bash
# 查看Agent状态
docker-compose ps

# 查看日志
docker-compose logs -f agent_name

# 重启Agent
docker-compose restart agent_name
```

#### 问题3：数据分析不准确
**症状**:
- 买卖信号异常
- 评分不合理
- 建议不符合预期

**排查步骤**:
1. 检查数据源
2. 验证分析算法
3. 检查参数配置

**解决方案**:
```bash
# 检查数据文件
docker exec -it postgres psql -U stock_user -d stock_system -c "SELECT * FROM market_data LIMIT 10;"

# 调整分析参数
vi config/agents/technical_analyzer.json
```

### 系统恢复步骤

#### 轻度故障
```
症状: 单个Agent异常
处理: 重启对应Agent
命令: docker-compose restart agent_name
时间: < 1分钟
```

#### 中度故障
```
症状: 多个Agent异常
处理: 重启整个系统
命令: docker-compose down && docker-compose up -d
时间: < 3分钟
```

#### 重度故障
```
症状: 系统完全崩溃
处理: 恢复备份配置
步骤:
1. 停止所有服务
2. 恢复配置文件
3. 恢复数据备份
4. 重新启动系统
时间: < 10分钟
```

## 常见问题

### Q1: 系统适合什么类型的投资者？
**A**: 系统专为**散户超短线（1-3天）**投资者设计，适合：
- 关注市场热点的投资者
- 喜欢技术分析的投资者
- 需要风险控制的投资者
- 希望自动化辅助的投资者

### Q2: 系统的准确率如何？
**A**: 系统准确率受多种因素影响：
- **数据质量**: 依赖数据源的准确性
- **市场环境**: 不同市场表现不同
- **参数设置**: 需要根据个人风格调整
- **建议**: 先用模拟盘测试，再实盘操作

### Q3: 需要多少资金？
**A**: 系统支持各种资金规模：
- **小资金**: 1-5万元，建议单只股票≤30%
- **中资金**: 5-20万元，建议分散投资
- **大资金**: 20万元以上，建议分批操作

### Q4: 如何提高收益？
**A**: 建议策略：
1. **严格止损**: 亏损8%立即止损
2. **分批建仓**: 不要一次性满仓
3. **关注热点**: 跟随市场主线
4. **控制情绪**: 避免追涨杀跌
5. **持续学习**: 总结经验教训

### Q5: 系统有哪些限制？
**A**: 当前版本限制：
- 数据源依赖外部API
- 分析基于历史数据
- 需要人工最终确认
- 实盘交易需要券商接口

### Q6: 如何获得技术支持？
**A**: 支持渠道：
- **钉钉群**: 实时在线支持
- **日志文件**: 自助排查问题
- **文档查询**: 参考使用手册
- **系统报告**: 查看状态信息
